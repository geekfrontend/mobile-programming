### ğŸ”¹ Array & Object

```ts
let numbers: number[] = [1, 2, 3, 4, 5];

let student: { name: string; age: number } = {
  name: "Michael",
  age: 20,
};
```

Array digunakan untuk menyimpan banyak nilai, sedangkan Object menyimpan pasangan `key-value`.

---

### ğŸ”¹ Interface

`interface` digunakan untuk mendefinisikan struktur data agar lebih konsisten.  

```ts
interface User {
  name: string;
  age: number;
  isActive: boolean;
}

const user1: User = {
  name: "Sarah",
  age: 22,
  isActive: true,
};
```

---

### ğŸ”¹ Type Alias vs Interface

Selain `interface`, kita juga bisa menggunakan `type`:  

```ts
type Product = {
  name: string;
  price: number;
  stock: number;
};

const book: Product = {
  name: "Learning React Native",
  price: 75000,
  stock: 10,
};
```

**Perbedaan utama:**  
- `interface` bisa di-*extend* (pewarisan).  
- `type` lebih fleksibel (bisa union/intersection).  

---

### ğŸ”¹ Generic

Generic membuat fungsi/kelas lebih fleksibel tanpa kehilangan keamanan tipe.  

```ts
function wrap<T>(value: T): T[] {
  return [value];
}

console.log(wrap<string>("Hello"));
console.log(wrap<number>(123));
```

### ğŸ”¹ Enum

`enum` digunakan untuk mendefinisikan kumpulan nilai yang tetap.  

```ts
enum Role {
  ADMIN,
  USER,
  GUEST,
}

let role: Role = Role.ADMIN;
console.log(role); // 0
```
